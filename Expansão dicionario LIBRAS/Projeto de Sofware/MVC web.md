O padrão MVC web foi escolhido pela semelhança para com o padrão 3 camadas, onde há a separação do cliente, da lógica e do banco de dados enquanto ainda mantém as características do padrão MVC, que permite que a aplicação assuma diferentes formas sem necessidade de reescrever código. Mais especificamente, o isolamento da "visão" permite que o sistema possa assumir diferentes interfaces, se adaptando a clientes que acessem o sistema de diferentes dispositivos. Da mesma forma, o isolamento dos "controladores" permitem que o sistema atue de formas diferentes ao ser acessado por diferentes usuários (Visitantes, Revisores e Administradores).
